## Context Bridging: Flexible Control Layouts

The `ef-controls` element bridges playback context from a target element to controls placed anywhere in the DOM, enabling flexible UI layouts that aren't constrained by DOM hierarchy.

### Engineering Challenges with Direct Nesting

Using control elements directly nested within preview elements presents these engineering challenges: Control elements must be descendants of the preview to access playback context, which constrains layout flexibility. UI designs often require controls in separate areas—sticky footers, sidebars, modals, or different screens—but direct nesting prevents this. Portal-based rendering patterns (React portals, Vue teleports) break context access when controls are rendered outside the component tree.

### The Solution: Context Bridging

The `ef-controls` element bridges context from a target preview to controls placed anywhere in the DOM. Controls inside `ef-controls` receive the same playback context as if they were nested directly in the preview, enabling flexible layouts without sacrificing functionality.

**Architectural advantages:**
- **Layout flexibility:** Controls can be placed anywhere in the DOM, not just as preview descendants
- **Portal support:** Works with React portals, Vue teleports, and similar rendering patterns
- **Multi-location control:** Multiple control sets can target the same preview from different locations
- **Real-time synchronization:** Context updates flow in both directions with minimal latency
- **No performance overhead:** Context bridging adds minimal overhead, maintaining responsive interactions

### Benefits for Your Application

**Flexible layouts:** Build interfaces where the video preview is in the main content area and controls are in a sticky footer, sidebar, or modal dialog. Position controls based on your design needs, not DOM constraints.

**Portal-based rendering:** Use React portals or Vue teleports to render controls outside the normal component tree while maintaining full functionality. Controls remain connected to the preview through context bridging.

**Multi-monitor setups:** Build applications where the video preview is on one screen and the control interface is on another. All controls stay synchronized through a single context bridge.

**Separation of concerns:** Keep preview and control logic separate in your component structure while maintaining tight synchronization. This improves code organization and maintainability.

Controls placed outside the preview work identically to nested controls:

```html
<ef-preview id="my-preview">...</ef-preview>

<ef-controls target="my-preview">
  <ef-toggle-play>...</ef-toggle-play>
  <ef-scrubber></ef-scrubber>
</ef-controls>
```

Without context bridging, controls must be nested inside the preview, constraining layout options. With `ef-controls`, controls can be placed anywhere while maintaining full functionality.
