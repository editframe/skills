2. Benchmark

A: All main thread — toDataURL + SVG + btoa + load (baseline)
F: ImageBitmap → 1 worker JPEG encode + SVG + btoa (proven best single-frame)
H: Pipelined JPEG — 2 workers alternating frames, measures throughput over 20-frame batch
I: Parallel JPEG — 1 encode worker per canvas + assembly worker, all encode simultaneously
J: Pipelined + Parallel — per-canvas parallel encoding with frame pipelining (H+I combined)
K: Parallel JPEG + Blob URL — skip btoa, use blob URL for SVG instead of data URI
L: Parallel JPEG + createImageBitmap(SVG) — decode SVG entirely in worker
M: Parallel JPEG + Resize — encode at display resolution, not source resolution
N: Parallel WebP — same as I but with image/webp encoding
O: Pipelined + Parallel + Resize — the ultimate: J+M combined
P: Native drawElementImage — Chrome experimental API, no SVG pipeline at all

H, J, and O report per-frame throughput averaged over a 20-frame batch. All others report per-frame latency.
