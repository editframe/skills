# Responsive Layouts

This guide demonstrates how to create responsive video layouts that adapt to different screen sizes and aspect ratios while maintaining proper video composition.

## Understanding Aspect Ratios

When creating videos, the root timegroup's dimensions define the output aspect ratio. For responsive displays, use `<ef-fit-scale>` to scale the content to fit the viewer's screen.

## Common Aspect Ratios

### 16:9 (Standard HD)

```html
<ef-timegroup mode="contain" class="w-[1920px] h-[1080px]">
  <!-- 1920x1080 = 16:9 -->
</ef-timegroup>
```

### 9:16 (Vertical/Mobile)

```html
<ef-timegroup mode="contain" class="w-[1080px] h-[1920px]">
  <!-- 1080x1920 = 9:16 (portrait) -->
</ef-timegroup>
```

### 1:1 (Square)

```html
<ef-timegroup mode="contain" class="w-[1080px] h-[1080px]">
  <!-- 1080x1080 = 1:1 (square for social media) -->
</ef-timegroup>
```

### 4:5 (Instagram)

```html
<ef-timegroup mode="contain" class="w-[1080px] h-[1350px]">
  <!-- 1080x1350 = 4:5 -->
</ef-timegroup>
```

## Mobile-Friendly Player

Create a player that works well on mobile devices:

```html
<style>
  .mobile-player {
    width: 100%;
    max-width: 100vw;
    height: 100vh;
    background: black;
  }
  
  @media (orientation: portrait) {
    .video-content {
      width: 1080px;
      height: 1920px;
    }
  }
  
  @media (orientation: landscape) {
    .video-content {
      width: 1920px;
      height: 1080px;
    }
  }
</style>

<ef-fit-scale class="mobile-player">
  <ef-timegroup mode="contain" class="video-content bg-black">
    <ef-video src="/video.mp4" class="w-full h-full object-contain"></ef-video>
  </ef-timegroup>
</ef-fit-scale>
```

## Container Queries (Modern Browsers)

Use container queries for ultra-responsive layouts:

```html
<style>
  .responsive-container {
    container-type: size;
    container-name: video-player;
  }
  
  @container video-player (max-width: 600px) {
    .title {
      font-size: 2rem;
    }
    .controls {
      flex-direction: column;
    }
  }
  
  @container video-player (min-width: 601px) {
    .title {
      font-size: 4rem;
    }
    .controls {
      flex-direction: row;
    }
  }
</style>

<div class="responsive-container">
  <ef-fit-scale style="width: 100%; height: 500px;">
    <ef-timegroup mode="contain" class="w-[1920px] h-[1080px]">

    </ef-timegroup>
  </ef-fit-scale>
</div>
```

## Maintaining Aspect Ratio

### 16:9 Container

Create a container that always maintains 16:9 aspect ratio:

```html
<style>
  .aspect-16-9 {
    position: relative;
    width: 100%;
    padding-bottom: 56.25%; /* 9/16 = 0.5625 = 56.25% */
  }
  
  .aspect-16-9 > ef-fit-scale {
    position: absolute;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
  }
</style>

<div class="aspect-16-9">
  <ef-fit-scale>
    <ef-timegroup mode="contain" class="w-[1920px] h-[1080px]">
      <ef-video src="/video.mp4"></ef-video>
    </ef-timegroup>
  </ef-fit-scale>
</div>
```

### Responsive Square

## Fullscreen Support

Add fullscreen capability to your player:

```html
<div id="player-container">
  <ef-preview>
    <ef-timegroup mode="contain" class="w-[1920px] h-[1080px]">
      <ef-video src="/video.mp4"></ef-video>
    </ef-timegroup>
    
    <div class="controls">
      <ef-toggle-play>...</ef-toggle-play>
      <ef-scrubber></ef-scrubber>
      <button onclick="toggleFullscreen()">⛶ Fullscreen</button>
    </div>
  </ef-preview>
</div>

<style>
  #player-container {
    width: 100%;
    max-width: 1200px;
    margin: 0 auto;
  }
  
  #player-container:fullscreen {
    width: 100vw;
    height: 100vh;
  }
  
  #player-container:fullscreen ef-fit-scale {
    width: 100%;
    height: 100%;
  }
</style>

<script>
  function toggleFullscreen() {
    const container = document.getElementById('player-container');
    if (!document.fullscreenElement) {
      container.requestFullscreen();
    } else {
      document.exitFullscreen();
    }
  }
</script>
```

## React Responsive Example

```tsx
import { useEffect, useState, useRef } from 'react';
import { FitScale, Timegroup, Video } from '@editframe/react';

export function ResponsivePlayer() {
  const containerRef = useRef<HTMLDivElement>(null);
  const [dimensions, setDimensions] = useState({ width: 1920, height: 1080 });
  
  useEffect(() => {
    const updateDimensions = () => {
      if (containerRef.current) {
        const { clientWidth } = containerRef.current;
        // Adjust composition based on available width
        if (clientWidth < 600) {
          setDimensions({ width: 1080, height: 1920 }); // Vertical
        } else {
          setDimensions({ width: 1920, height: 1080 }); // Horizontal
        }
      }
    };
    
    updateDimensions();
    window.addEventListener('resize', updateDimensions);
    return () => window.removeEventListener('resize', updateDimensions);
  }, []);
  
  return (
    <div ref={containerRef} style={{ width: '100%', height: '500px' }}>
      <FitScale style={{ width: '100%', height: '100%' }}>
        <Timegroup 
          mode="contain" 
          className={`w-[${dimensions.width}px] h-[${dimensions.height}px] bg-black`}
        >
          <Video src="/video.mp4" className="w-full h-full object-contain" />
        </Timegroup>
      </FitScale>
    </div>
  );
}
```

## Safe Areas

```css

  padding: env(safe-area-inset-top) 
           env(safe-area-inset-right) 
           env(safe-area-inset-bottom) 
           env(safe-area-inset-left);

.safe-area-content {
  /* Keep important content within safe areas */

           max(20px, env(safe-area-inset-right)) 
           max(20px, env(safe-area-inset-bottom)) 
           max(20px, env(safe-area-inset-left));
}
```

## Best Practices

### 1. Define Composition Size First

Always define your composition dimensions based on the target output:
- 1920x1080 for HD horizontal videos
- 1080x1920 for mobile/vertical videos
- 1080x1080 for square social media

### 2. Use Fit-Scale for Display

Wrap in `<ef-fit-scale>` to adapt to any display size while maintaining aspect ratio.

### 3. Design for Mobile First

When building responsive experiences:
- Design for the smallest screen first
- Scale up for larger screens
- Test on actual devices

### 4. Consider Touch Targets

On mobile, ensure controls are large enough:
- Minimum 44x44px for touch targets
- Add spacing between controls
- Use larger fonts for readability

### 5. Test Multiple Orientations

Test both portrait and landscape orientations on mobile devices.
