# Image

The `<ef-image>` element is a custom HTML element that extends the functionality of the standard HTML image element. It provides essential integration with the Editframe rendering system.

## Basic Usage

<Demonstration layout="horizontal" defaultSourceVisible alwaysShowSource>
  <Timegroup mode="fixed" duration="5s" className="w-[960px] h-[540px] bg-black flex overflow-hidden">
    <Image className="w-full h-full object-cover absolute top-0 left-0" src="https://assets.editframe.com/bridge.jpg"/>
  </Timegroup>
</Demonstration>

## Attributes

<PropertyDocList>
  <PropertyDoc name="assetId" type="string" domReadable domWritable htmlAttribute="asset-id">
    The asset ID of the image.

    An asset ID is a unique identifier for a media file that has been uploaded to Editframe. This is documented in the [Processing Files](/docs/processing-files/image-files) section.
  </PropertyDoc>

  <PropertyDoc name="src" type="string" domReadable domWritable htmlAttribute="src">
    The source URL/path of the image.

    <div className="bg-blue-100 dark:bg-blue-900/30 text-blue-900 dark:text-blue-100 p-2 mb-2 rounded border border-blue-200 dark:border-blue-800">
      ⚠️ This property is not yet supported in all circumstances. Providing an arbitrary URL or path will not work unless the server is configured to respond with a precise, not-yet-documented, format.

      Demonstration projects with such a server are available via `npm create @editframe@beta`. These are not intended for production use. When complete, a specialized server should not be required, though there will be specific encoding/muxing limitations.

      Instead upload media to Editframe and use the `assetId` property.
    </div>

    <div className="bg-orange-100 dark:bg-orange-900/30 text-orange-900 dark:text-orange-100 p-2 rounded border border-orange-200 dark:border-orange-800">
      🚫 This property should only be used for development, testing, and previewing media to end-users in real-time. When submitting a render job, all media should be uploaded to Editframe and the `assetId` property should be used. Our render servers will render in parallel, and in order to operate efficiently we need to be able to load only the slice of media that is needed to render a small segment of the timeline.
    </div>
  </PropertyDoc>

  <PropertyDoc
    name="duration"
    domReadable
    domWritable
    htmlAttribute
    type="timestring"
    defaultValue=""
  >
    A fixed duration for the image. Typically, an image will expand to fill the duration of its parent timegroup. However, sometimes you may want to display an image for a fixed duration, such is in a simple slideshow inside a timegroup.
  </PropertyDoc>
  
</PropertyDocList>

## Demonstrations

### Single image 

This example demonstrates a single image being displayed for 5 seconds. Because all videos require a root timegroup, this is the simplest way to display a single image.

<Demonstration layout="horizontal" defaultSourceVisible alwaysShowSource>
  <Timegroup mode="fixed" duration="5s" className="w-[960px] h-[540px] bg-black overflow-hidden">
    <Image className="w-full h-full object-contain object-center absolute top-0 left-0" src="https://assets.editframe.com/bridge.jpg"/>
  </Timegroup>
</Demonstration>

### Image With Fixed Duration

In that example, the root timegroup was set to a fixed duration of 5 seconds. By default, an image will expand temporally to fill it's parent timegroup.

We can invert this relationship by using the `contain` mode on the timegroup and giving the image a fixed duration.

<Demonstration layout="horizontal" defaultSourceVisible alwaysShowSource>
  <Timegroup mode="contain" className="w-[960px] h-[540px] bg-black overflow-hidden">
    <Image duration="5s" className="w-full h-full object-contain object-center absolute top-0 left-0" src="https://assets.editframe.com/bridge.jpg"/>
  </Timegroup>
</Demonstration>

### Simple Image Sequence

We can create a simple slideshow of images by using the `sequence` mode on the timegroup, and giving each image a fixed duration.

<Demonstration layout="horizontal" defaultSourceVisible alwaysShowSource>
  <Timegroup mode="sequence" className="w-[540px] h-[540px] bg-black overflow-hidden">
    <Image duration="5s" className="w-full h-full object-contain object-center absolute top-0 left-0" src="https://assets.editframe.com/dog.jpg"/>
    <Image duration="5s" className="w-full h-full object-contain object-center absolute top-0 left-0" src="https://assets.editframe.com/meme.jpg"/>
  </Timegroup>
</Demonstration>

### Rich Slideshow

In practice, you may want to include more content on each "slide". For those cases, it would be better to wrap each image in a timegroup to contain the other content.

<Demonstration layout="horizontal" defaultSourceVisible alwaysShowSource>
  <Timegroup id="rich-slideshow" mode="sequence" className="w-[540px] h-[540px] bg-black overflow-hidden">
    <Timegroup mode="fixed" duration="5s" className="w-full h-full bg-black">
      <Image className="w-full h-full object-contain object-center absolute top-0 left-0" src="https://assets.editframe.com/dog.jpg"/>
      <h1 className="bg-blue-900 text-white text-center absolute p-2 bottom-0 left-0 w-full z-10">Breaking news: Handsome boy!</h1>
    </Timegroup>
    <Timegroup mode="fixed" duration="5s" className="w-full h-full bg-black grid grid-cols-[1fr_1fr] grid-rows-[1fr_1fr]">
      <Image className="w-full h-full object-contain object-center absolute top-0 left-0" src="https://assets.editframe.com/meme.jpg"/>
      <h1 className="text-black col-start-2 row-start-1 w-1/4 self-center pl-8 z-10">Image Sequence</h1>
      <h1 className="text-black col-start-2 row-start-2 w-1/4 self-center pl-8 z-10">Timegroup Sequence w/Images</h1>
    </Timegroup>
  </Timegroup>
</Demonstration>
