## Demonstration

### Basic Usage

A surface element displays content from a target video element. Surfaces are the preferred element to use when you want to show the same video source more than once in synchronization, using significantly fewer resources than duplicate video elements:

<Demonstration layout="horizontal" defaultSourceVisible alwaysShowSource>
  <Timegroup mode="contain" className="w-[720px] h-[480px] flex items-center justify-center bg-black">
    <Video id="source-video" src="https://assets.editframe.com/bars-n-tone.mp4" className="w-1/2 h-full object-contain" />
    <Surface target="source-video" className="w-1/2" />
  </Timegroup>
</Demonstration>

### Multiple Surfaces from Single Source

Multiple surfaces from a single video source are more efficient than multiple video elements:

<Demonstration layout="horizontal" defaultSourceVisible alwaysShowSource>
  <Timegroup mode="contain" className="w-[720px] h-[480px] flex items-center justify-center bg-black">
    <Video id="main-video" src="https://assets.editframe.com/bars-n-tone.mp4" className="w-1/3 h-full object-contain" />
    <Surface target="main-video" className="w-1/3" />
    <Surface target="main-video" className="w-1/3" />
  </Timegroup>
</Demonstration>

### Efficient Filter Styling

Surfaces can each have different CSS filter styles applied efficiently from a single video source:

<Demonstration layout="horizontal" defaultSourceVisible alwaysShowSource>
  <Timegroup mode="contain" className="w-[720px] h-[480px] grid grid-cols-3 gap-2 p-2 bg-black">
    <Video id="filter-video" src="https://assets.editframe.com/bars-n-tone.mp4" className="w-full h-full object-cover" />
    <Surface target="filter-video" className="w-full h-full object-cover grayscale" />
    <Surface target="filter-video" className="w-full h-full object-cover sepia" />
    <Surface target="filter-video" className="w-full h-full object-cover blur-sm" />
    <Surface target="filter-video" className="w-full h-full object-cover saturate-200" />
    <Surface target="filter-video" className="w-full h-full object-cover hue-rotate-90" />
    <Surface target="filter-video" className="w-full h-full object-cover brightness-150" />
    <Surface target="filter-video" className="w-full h-full object-cover contrast-200" />
    <Surface target="filter-video" className="w-full h-full object-cover invert" />
  </Timegroup>
</Demonstration>

## Attributes

<PropertyDocList>
  <PropertyDoc name="target" type="string" domReadable domWritable htmlAttribute="target">
    The ID of the element to copy content from, typically an `ef-video` element.

    <div className="bg-blue-100 text-blue-900 p-2 mb-2 rounded">
      💡 The target element must have an `id` attribute. The surface will look for an element with that ID and copy its canvas content.
    </div>

    <div className="bg-green-100 text-green-900 p-2 rounded">
      ✅ **Performance Benefit**: Multiple surfaces targeting the same video element use significantly fewer resources than multiple `ef-video` elements.
    </div>
  </PropertyDoc>

  <PropertyDoc name="targetElement" type="Element | null" domReadable>
    The resolved target element that this surface is copying from.

    This property is automatically set when the `target` attribute is resolved to an actual DOM element.
  </PropertyDoc>

  <PropertyDoc name="startTimeMs" type="number" domReadable>
    The start time of the surface in milliseconds. This time is relative to the timeline described by the root timegroup the surface is contained within.

    <div className="bg-blue-100 text-blue-900 p-2 rounded">
      💡 Surface elements inherit their timing from their target element and containing timegroup.
    </div>
  </PropertyDoc>

  <PropertyDoc name="endTimeMs" type="number" domReadable>
    The end time of the surface in milliseconds. This time is relative to the timeline described by the root timegroup the surface is contained within.
  </PropertyDoc>

  <PropertyDoc name="durationMs" type="number" domReadable>
    The duration of the surface in milliseconds. This is equivalent to the difference between `endTimeMs` and `startTimeMs`.
  </PropertyDoc>

  <PropertyDoc name="currentTimeMs" type="number" domReadable>
    The current time of the surface in milliseconds. This time is the time within the timeline of the root timegroup the surface is contained within.
  </PropertyDoc>

  <PropertyDoc name="rootTimegroup" type="<ef-timegroup> | null" domReadable>
    The outermost timegroup element that contains the surface.

    The surface will prefer to use the target element's root timegroup if available, falling back to its own containing timegroup.
  </PropertyDoc>
</PropertyDocList>

## Technical Notes

<div className="bg-yellow-100 text-yellow-900 p-4 rounded">
  **Frame Synchronization**: Surfaces automatically wait for their target element to complete frame rendering before copying, ensuring synchronized playback across all surfaces.
</div>

<div className="bg-blue-100 text-blue-900 p-4 rounded">
  **Canvas Mirroring**: Surfaces create pixel-perfect copies of the target's canvas content, with CSS handling any layout scaling needed.
</div>
