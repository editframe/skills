# CSS Animations

 CSS animations are a powerful tool for creating dynamic and engaging video compositions. In this guide, we'll cover how to use CSS animations in Editframe video compositions to create seamless visual effects.

## Overview
Now, let's explain the usage of CSS animations in Editframe video compositions:

1. Defining animations:

- CSS animations are defined using the @keyframes rule. In the example, we've defined three animations: `slide-up`, `fade-in`, and `spin`.

2. Applying animations:
- Animations are applied to elements using the animation property. In Editframe, you can apply animations to any element within an ef-timegroup.

3. Using var(--ef-duration):
- Editframe provides a CSS variable `--ef-duration` that represents the duration of the parent ef-timegroup. This allows you to synchronize your animations with the timegroup's duration.
Example:
```css
.animated-element {
    animation: var(--ef-duration) slide-up;
}
```
This will make the `slide-up` animation last for the entire duration of the parent timegroup.
4. Combining multiple animations:
- You can apply multiple animations to a single element by separating them with commas. Each animation can have its own duration and delay.
Example:
```css
.animated-element {
    animation: 
        var(--ef-duration) slide-up,
        calc(var(--ef-duration) * 0.5) fade-in,
        var(--ef-duration) spin calc(var(--ef-duration) * 0.5);
}
```
This applies three animations:

- `slide-up` for the full duration
- `fade-in` for half the duration
- `spin` for the full duration, starting halfway through


- Animation timing:
You can control when animations start and end within the timegroup using CSS animation properties like animation-delay and animation-duration.

By using these techniques, you can create complex, synchronized animations that work seamlessly within your Editframe video compositions. The var(--ef-duration) feature is particularly useful for ensuring that your animations align perfectly with the timing of your video elements.
