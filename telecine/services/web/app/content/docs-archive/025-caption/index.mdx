# Introduction

The `<ef-captions>` and `<ef-captions-active-word>` elements are custom components designed for use within the Editframe ecosystem. They work together to provide synchronized captions for audio or video content, with the ability to highlight the currently spoken word.

## Key Features

* **Synchronized Captions**: Displays captions in sync with audio or video playback.
* **Word Highlighting**: Uses `<ef-captions-active-word>` to highlight the current word being spoken.
* **Customizable**: Allows for styling to match your project's design.

## Attributes

* **target**: The ID of the audio or video element to synchronize with.

## Example

```html
<ef-audio id="caption-audio" src="assets/audio-sample.mp3"></ef-audio>
<ef-captions
  class="h-24 p-2 text-center text-4xl"
  target="caption-audio"
>
  <ef-captions-active-word
    style="
      animation: 0.1s bounce-in ease-out forwards;
      animation-play-state: paused;
    "
  ></ef-captions-active-word>
</ef-captions>
```

This will create a captions display for the audio with the ID "caption-audio". The captions will be centered, with a text size of 4xl, and the currently spoken word will be highlighted with a bounce-in animation.

## Styling

You can customize the appearance of the captions and active word using CSS classes and inline styles. The `class` attribute on `<ef-captions>` allows you to style the container, while the `style` attribute on `<ef-captions-active-word>` lets you define animations and styles for the highlighted word.

## Use Cases

The `<ef-captions>` and `<ef-captions-active-word>` elements are particularly useful in the following scenarios:

* Accessibility features for audio and video content
* Language learning applications
* Karaoke-style lyric displays
* Any project where synchronized text with audio or video enhances the user experience

By using these elements, you can create more accessible and engaging audio-visual experiences within your Editframe projects.
