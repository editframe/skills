## Reference

### Grouped Attributes

#### Data Source

<PropertyDocList>
  <PropertyDoc name="captionsData" type="Caption | null" domReadable domWritable>
    Direct captions data object. Takes priority over `captions-src` and `captions-script`.

    The data must conform to the Caption interface with `segments` and `word_segments` arrays. See the [Captions Data Format explanation](/docs/elements/captions/explanation/captions-data-format) for details.

    ```javascript
    const captions = document.querySelector('ef-captions');
    captions.captionsData = {
      segments: [
        { start: 0, end: 3, text: "First caption" }
      ],
      word_segments: [
        { start: 0, end: 0.5, text: "First" },
        { start: 0.5, end: 1.0, text: "caption" }
      ]
    };
    ```
  </PropertyDoc>

  <PropertyDoc name="captionsScript" type="string" domReadable domWritable htmlAttribute="captions-script">
    ID of a `<script type="application/json">` element containing captions data. Do not include the # character.

    ```html
    <script type="application/json" id="my-captions">
    {
      "segments": [
        { "start": 0, "end": 3, "text": "First caption" }
      ],
      "word_segments": [
        { "start": 0, "end": 0.5, "text": "First" },
        { "start": 0.5, "end": 1.0, "text": "caption" }
      ]
    }
    </script>
    <ef-captions captions-script="my-captions">
      <ef-captions-segment />
    </ef-captions>
    ```

    See the [Captions Data Format explanation](/docs/elements/captions/explanation/captions-data-format) for the required JSON structure.
  </PropertyDoc>

  <PropertyDoc name="captionsSrc" type="string" domReadable domWritable htmlAttribute="captions-src">
    URL or path to an external JSON file containing captions data.

    ```html
    <ef-captions captions-src="/path/to/captions.json">
      <ef-captions-segment />
    </ef-captions>
    ```

    See the [Captions Data Format explanation](/docs/elements/captions/explanation/captions-data-format) for the required JSON structure.
  </PropertyDoc>

  <PropertyDoc name="assetId" type="string" domReadable domWritable htmlAttribute="asset-id">
    The asset ID of a caption file uploaded to Editframe.

    An asset ID is a unique identifier for a caption file that has been uploaded to Editframe. This is documented in the [Processing Files](/docs/processing-files/audio-video-files) section.

    When using `assetId`, captions are automatically loaded from the uploaded caption file.
  </PropertyDoc>
</PropertyDocList>

#### Synchronization

<PropertyDocList>
  <PropertyDoc name="target" type="string" domReadable domWritable htmlAttribute="target">
    The ID of a video or audio element to synchronize with. Do not include the # character.

    When set, captions automatically synchronize with the target element's playback. When omitted, captions run on their own timeline within the parent timegroup.

    ```html
    <ef-video id="my-video" src="video.mp4" />
    <ef-captions target="my-video">
      <ef-captions-active-word />
    </ef-captions>
    ```

    See the [Temporal Synchronization explanation](/docs/elements/captions/explanation/temporal-synchronization) for details on how synchronization works.
  </PropertyDoc>
</PropertyDocList>

#### Time Coordinates

<PropertyDocList>
  <PropertyDoc name="durationMs" type="number" domReadable>
    Duration of the captions in milliseconds, calculated from the maximum end time of all segments and word_segments in the captions data.

    When captions have custom data (via `captionsData`, `captionsScript`, or `captionsSrc`), this property reflects the duration of that data. Otherwise, it reflects the duration of the transcription fragment loaded from the target element.
  </PropertyDoc>

  <PropertyDoc name="startTimeMs" type="number" domReadable>
    The start time of the captions in milliseconds. This time is relative to the timeline described by the root timegroup the captions are contained within, or 0 if the captions are standalone (no timegroup).
  </PropertyDoc>

  <PropertyDoc name="endTimeMs" type="number" domReadable>
    The end time of the captions in milliseconds. This time is relative to the timeline described by the root timegroup the captions are contained within, or the captions' duration if standalone (no timegroup).
  </PropertyDoc>

  <PropertyDoc name="currentTimeMs" type="number" domReadable>
    The current time of the captions in milliseconds. This time is the time within the timeline of the root timegroup the captions are contained within, or the captions' own timeline if standalone (no timegroup).
  </PropertyDoc>

  <PropertyDoc name="ownCurrentTimeMs" type="number" domReadable>
    The current time of the captions in milliseconds. This time is scoped directly to the captions element itself and is synchronized with the timegroup or target element.
  </PropertyDoc>

  <PropertyDoc name="currentSourceTimeMs" type="number" domReadable>
    The current time of the captions in milliseconds scoped to the source captions timeline. If no `trimStart` or `sourceIn` are set, this property is equivalent to `ownCurrentTimeMs`.
  </PropertyDoc>

  <PropertyDoc name="offset" type="timestring" domReadable domWritable htmlAttribute="offset">
    Offset of the captions within its parent timegroup. This can be set in seconds or milliseconds (e.g., "10s" or "10000ms").
  </PropertyDoc>

  <PropertyDoc name="duration" type="timestring" domReadable domWritable htmlAttribute="duration">
    Explicit duration override. This can be set in seconds or milliseconds (e.g., "10s" or "10000ms").
  </PropertyDoc>
</PropertyDocList>

#### Trimming & Editing

<PropertyDocList>
  <PropertyDoc name="trimStart" type="timestring" domReadable domWritable htmlAttribute="trimstart">
    A duration of time to trim off from the start of the captions.

    This time is relative to the start of the captions. Setting trimStart to 10s will result in the captions starting 10 seconds into the source captions data.

    This property is intended to be used with the `trimEnd` property.
  </PropertyDoc>

  <PropertyDoc name="trimEnd" type="timestring" domReadable domWritable htmlAttribute="trimend">
    A duration of time to trim off from the end of the captions.

    This time is relative to the end of the captions. Setting trimEnd to 10s will result in the captions ending 10 seconds before the end of the source captions data.

    This property is intended to be used with the `trimStart` property.
  </PropertyDoc>

  <PropertyDoc name="sourceIn" type="timestring" domReadable domWritable htmlAttribute="sourcein">
    Absolute source start time. This can be set in seconds or milliseconds (e.g., "10s" or "10000ms").

    Setting sourceIn to 10s will result in the captions starting at exactly 10 seconds into the source captions data.

    This property is intended to be used with the `sourceOut` property.
  </PropertyDoc>

  <PropertyDoc name="sourceOut" type="timestring" domReadable domWritable htmlAttribute="sourceout">
    Absolute source end time. This can be set in seconds or milliseconds (e.g., "10s" or "10000ms").

    Setting sourceOut to 10s will result in the captions ending at exactly 10 seconds into the source captions data.

    This property is intended to be used with the `sourceIn` property.
  </PropertyDoc>
</PropertyDocList>

    The immediate containing timegroup of the captions element.
  </PropertyDoc>

  <PropertyDoc name="rootTimegroup" type="<ef-timegroup> | null" domReadable>
    The root timeline timegroup containing the captions element.
  </PropertyDoc>
</PropertyDocList>

#### Styling

<PropertyDocList>
  <PropertyDoc name="wordStyle" type="string" domReadable domWritable htmlAttribute="word-style">
    CSS styling for active word elements. This property allows you to apply inline styles to word elements.

    ```html
    <ef-captions word-style="color: yellow; font-weight: bold;">
      <ef-captions-active-word />
    </ef-captions>
    ```
  </PropertyDoc>
</PropertyDocList>

#### Playback Control

<PropertyDocList>
  <PropertyDoc name="playing" type="boolean" domReadable>
    Whether playback is currently active. This property reflects the playback state of the root timegroup or target element.
  </PropertyDoc>

  <PropertyDoc name="loop" type="boolean" domReadable domWritable htmlAttribute="loop">
    Whether to loop playback. When set to true, captions will restart from the beginning when they reach the end.
  </PropertyDoc>
</PropertyDocList>

### Child Elements

The captions element uses child elements to display different parts of the captions:

#### `<ef-captions-segment>`

Displays the complete caption segment (phrase or sentence) that is currently active.

```html
<ef-captions>
  <ef-captions-segment />
</ef-captions>
```

This element shows the full text of the segment that matches the current playback time.

#### `<ef-captions-active-word>`

Displays the individual word that is currently being spoken.

```html
<ef-captions>
  <ef-captions-active-word />
</ef-captions>
```

This element provides access to the `--ef-word-seed` CSS variable for creating varied animations. See the [CSS Animation Variables explanation](/docs/elements/captions/explanation/css-animation-variables) for details.

#### `<ef-captions-before-active-word>`

Displays all words that come before the currently active word within the current segment.

```html
<ef-captions>
  <ef-captions-before-active-word />
  <ef-captions-active-word />
  <ef-captions-after-active-word />
</ef-captions>
```

This element is useful for creating context-aware displays where completed words are shown differently from the active word.

#### `<ef-captions-after-active-word>`

Displays all words that come after the currently active word within the current segment.

```html
<ef-captions>
  <ef-captions-before-active-word />
  <ef-captions-active-word />
  <ef-captions-after-active-word />
</ef-captions>
```

This element is useful for creating context-aware displays where upcoming words are shown differently from the active word.

### CSS Variables

Captions elements provide CSS variables for creating dynamic animations and styling:

#### Timeline Variables

Available on all caption elements:

- `--ef-duration`: Duration of the parent timegroup in milliseconds
- `--ef-progress`: Current progress as a number (0-1), not a CSS percentage. Stored as a number (not a percentage string) so it works correctly in CSS calculations like `calc(var(--ef-progress) * 100%)` without being converted to pixel values based on element dimensions.

#### Word-Specific Variables

Available on `<ef-captions-active-word>`:

- `--ef-word-seed`: A deterministic pseudo-random value (0-1) based on the word's position in the transcript

The `--ef-word-seed` variable enables creating varied animations where each word gets its own unique style, preventing repetitive-looking animations. See the [CSS Animation Variables explanation](/docs/elements/captions/explanation/css-animation-variables) for details and examples.
