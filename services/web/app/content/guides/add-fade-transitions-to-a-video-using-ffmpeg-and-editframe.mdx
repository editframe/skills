Transitions are key elements of video content. Well-deployed transitions like fades in and out can make your video content much more dynamic and engaging. Using a video editor to add these simple transitions can often be time-consuming, however—especially when you are adding these elements to multiple videos in a batch.

In this quick guide, you’ll learn how to use FFmpeg and Editframe to programmatically add fade-in and fade-out transitions to your videos. Use cases for this workflow include:

- Using fade transitions when stitching multiple videos together
- Smoothing out transitions between intros and outros
- Giving videos a more cinematic experience
- Generally improving transitions in videos

Let’s get started!

## Part 1: Using FFmpeg

First, we’ll walk through this workflow using FFmpeg.

### Required Tools

- Sample video file overlay (provided above)
- [**FFmpeg**](https://ffmpeg.org/download.html): (You’ll need to install FFmpeg and set up the appropriate environment variables before beginning this tutorial)

Here is the command to add a fade in transition to a video using FFmpeg:

```shell
ffmpeg -i video.mp4 -vf "fade=t=in:st=0:d=3" -c:a copy fadein.mp4
```

Let’s take a look at what this command is doing.

- In this line, we import the video input file:

```shell
ffmpeg -i video.mp4
```

- Here, we add the transition fade-in effect using the video effects filter `-vf`. `fade=t=in` is the name of the transition, `st=0` is the start time for the fade (0 seconds), and `d=3` is the duration of the effect (3 seconds):

```shell
-vf "fade=t=in:st=0:d=3"
```

- In this line, we copy the streams into the `fadein.mp4` video file:

```shell
-c:a copy fadein.mp4
```

### Adding a fade-out transition (FFmpeg)

Here is the command to add a fade-out transition with FFmpeg:

```shell
ffmpeg -i video.mp4 -vf "fade=t=out:st=4.06:d=3" -c:a copy fadeout.mp4
```

Once again, let’s step through each piece of the command.

- In this line, we import the video input file:

```shell
ffmpeg -i video.mp4
```

- Here, we add the transition fade-in effect using the video effects filter `-vf`. `fade=t=out` is the name of the transition, `st=4.06` is the start time at 4.06 seconds (total video duration - fade-out effect duration) and `d=3` is the duration of the effect (3 seconds).

```shell
-vf "fade=t=out:st=4.06:d=3"
```

- In this line, we copy the streams into the `fadeout.mp4` video file:

```shell
-c:a copy fadeout.mp4
```

## Part 2: Using Editframe

Now let’s perform the same task using Editframe instead of FFmpeg.

### Required tools:

- **Node.js** installed on your machine
- No need to have FFmpeg installed on your machine
- Editframe API Token (you can create an account from [this link](https://editframe.com/auth/register))

### Let’s get started:


1. Setup a new Node.js project using the Editframe CLI:

```shell
 npx @editframe/create@beta
```

2. Install the required dependencies:

```shell
cd add-fade-transition && npm install
```

3. Add assets to the project:

Add `video.mp4` to the `src/assets` folder.


4. Update the `index.html` file with the following code:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <script type="module" src="./src/index.js"></script>
    <link rel="stylesheet" href="./src/styles.css" />
    <style>
      @keyframes fade-out {
        from {
          opacity: 1;
        }
        to {
          opacity: 0;
        }
      }
    </style>
  </head>
  <body>
    <ef-timegroup mode="sequence" class="w-[1920px] h-[1080px]">
      <ef-timegroup mode="contain">
        <ef-video
          style="animation: 3s fade-out calc(var(--ef-duration) - 3s) forwards"
          id="video"
          class="w-full h-full object-cover"
          src="/assets/video.mp4"
        ></ef-video>
      </ef-timegroup>
    </ef-timegroup>
  </body>
</html>
```


Let’s walk through what the code in this file is doing. 

- In these lines, we import the `index.js` file and the `styles.css` file which contains the styles for the video and `@editframe/elements`:

```html
  <head>
    <meta charset="UTF-8" />
    <script type="module" src="./src/index.js"></script>
    <link rel="stylesheet" href="./src/styles.css" />
  </head>
```

- In these lines, we add `fade-out` CSS animation to use later on the video element:

```html
    <style>
      @keyframes fade-out {
        from {
          opacity: 1;
        }
        to {
          opacity: 0;
        }
      }
    </style>
```

- In this line, we create a new video composition using the `ef-timegroup` element. A vertical video (9:16) with a height of 1920px and a width of 1080px:

```html
<ef-timegroup
      mode="sequence"
      class="w-[1920px] h-[1080px]"
    >
    { /* Add elements here */ }
</ef-timegroup>
```

- In these lines, we set a new time group with a mode of `contain` to add the video:

```html
<ef-timegroup mode="contain">
        { /* Add elements here */ }
</ef-timegroup>
```

- In these lines, we import the video element:

```html
<ef-video
  style="animation: 3s fade-out calc(var(--ef-duration) - 3s) forwards"
  id="video"
  class="w-full h-full object-cover"
  src="/assets/video.mp4"
></ef-video>
   ```

- [`ef-video`](/docs/elements/video) is an Editframe element that allows you to add videos to your video composition.

Note: you can use the following code to add a fade-in transition to the video:

```html
<ef-video
  style="animation: 3s fade-in"
  id="video"
  class="w-full h-full object-cover"
  src="/assets/video.mp4"
></ef-video>
```
- Add the following code to the `src/index.html` file to create a fade-in animation: (in case you want to add a fade-in transition)

```html
<style>
@keyframes fade-in {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}
</style>
```

5. Preview the project:

```shell
npx vite .
```

6. Update the `.env` file with your Editframe API token:

```shell
EF_TOKEN="YOUR_API_TOKEN"
```

7. Render the video:

```shell
npx @editframe/cli render .
```
