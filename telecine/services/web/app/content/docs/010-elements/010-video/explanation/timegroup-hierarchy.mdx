## Timegroup Hierarchy

Video elements can exist within a hierarchy of timegroups that affects how time coordinates work. However, **timegroups are not required** - a standalone video element can be played directly without any timegroup wrapper.

### When Timegroups Are Required

Timegroups are needed when:
- Combining multiple elements (videos, text, graphics) in a composition
- Creating sequences of videos that play one after another
- Layering content with precise timing relationships
- Coordinating multiple elements on a shared timeline

### When Timegroups Are Optional

A standalone video can be used directly without a timegroup:
```jsx
<Video src="https://assets.editframe.com/video.mp4" />
```

In this case, `parentTimegroup` and `rootTimegroup` will both be `null`, and the video plays as a temporal element with its own timeline.

### parentTimegroup

The `parentTimegroup` property references the closest containing timegroup, or `null` if the video is not contained within any timegroup. This is important for:
- Understanding immediate containment
- Positioning relative to parent container
- Local timing context

### rootTimegroup

The `rootTimegroup` property references the outermost timegroup that defines the main timeline, or `null` if the video is standalone. This is important for:
- Global timeline positioning
- Understanding when your video appears in the overall composition
- Coordinating with other elements on the main timeline

**Example with timegroups:**
```jsx
<Timegroup>                          {/* Root timegroup */}
  <Timegroup mode="sequence">        {/* Parent timegroup */}
    <Video src="..." />              {/* Video element */}
    {/* parentTimegroup → sequence timegroup */}
    {/* rootTimegroup → root timegroup */}
  </Timegroup>
</Timegroup>
```

**Example without timegroups:**
```jsx
<Video src="https://assets.editframe.com/video.mp4" />
{/* parentTimegroup → null */}
{/* rootTimegroup → null */}
```

**Why this matters:**
- `startTimeMs` is relative to `rootTimegroup` when present, otherwise relative to the video's own timeline
- Nested sequences create local timelines that are then composed into the root timeline
- Understanding the hierarchy helps debug timing issues
- Standalone videos are simpler to use but don't support multi-element compositions

For more details on timegroups, see the [Timegroup documentation](/docs/elements/timegroup). For property details, see the [Video Element Reference](/docs/elements/video/reference#hierarchy).
