## How to Optimize Video Playback

**Problem:** You're concerned about performance, bandwidth usage, or loading times.

**Solution:** Configure buffering properties to control how much video is preloaded.

**Buffering configuration:**
```jsx
<Video 
  src="..."
  videoBufferDurationMs={10000}  // Buffer 10s ahead (default)
  maxVideoBufferFetches={2}       // Parallel segment fetches (default)
  enableVideoBuffering={true}     // Enable buffering (default)
/>
```

**Adjusting buffer duration:**
- Increase `videoBufferDurationMs` for smoother playback on slow connections (uses more bandwidth)
- Decrease for lower memory usage and faster initial load
- Default of 10 seconds works well for most cases

**Parallel fetching:**
- Increase `maxVideoBufferFetches` for faster buffering (may impact other network requests)
- Decrease if you need to prioritize other network activity
- Default of 2 concurrent fetches balances speed and resource usage

**Best practices:**
- For production rendering: Use `assetId` instead of `src` to enable parallel rendering
- For development/preview: Use `src` for convenience
- Browser caching handles repeat views automatically

For details on how the buffering system works, see the [JIT Transcoding Architecture explanation](/docs/elements/video/explanation/jit-transcoding).
