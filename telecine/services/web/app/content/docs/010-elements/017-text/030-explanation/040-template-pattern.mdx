## Template Pattern: Declarative Multi-Segment System

The text element uses a declarative template pattern that allows multiple visual segments per text unit (character, word, or line) without requiring JavaScript. This architecture addresses platform constraints that arise when creating layered text effects.

### The Problem: Platform Constraints in Layered Effects

Creating multiple visual elements per text unit presents these engineering challenges:

**Single-segment limitations:**
- Can't create layered effects (shadows, outlines, glows) with one element per unit
- Requires JavaScript to create multiple elements
- Complex to manage multiple element hierarchies
- Performance overhead from manual DOM manipulation

**Example:** To create a shadow effect, you'd need to manually duplicate each character, position elements, and manage timing - all in JavaScript.

### The Solution: Declarative Template Cloning

The text element uses HTML `<template>` elements that are cloned for each text unit:

**Template structure:**
- Place `<template>` inside `<Text>` element
- Add one or more `<TextSegment>` elements in the template
- Each character/word/line gets all segments from the template
- Segments are cloned and connected automatically

**How it works:**
1. Text is split into segments (characters, words, or lines)
2. Template content is cloned for each text segment
3. Each cloned segment receives the text content and timing properties
4. Segments are connected to DOM with proper hierarchy

**Example:** Template with shadow and main text:
```html
<ef-text split="char">
  <template>
    <ef-text-segment class="shadow" />
    <ef-text-segment class="main" />
  </template>
  Hello
</ef-text>
```

This creates two segments per character - one for shadow, one for main text.

### Benefits

**Declarative syntax:**
- No JavaScript required - works with pure HTML
- Text content can be set as text nodes
- Templates are standard HTML elements
- Easy to understand and maintain

**Multiple segments per unit:**
- Create shadows, outlines, glows, or any layered effect
- All segments share the same timing and stagger
- Automatic synchronization between layers
- No manual coordination needed

**Performance:**
- Browser-native template cloning is efficient
- Single pass through text content
- Minimal DOM manipulation overhead
- Scales well with long text

**Flexibility:**
- Any number of segments per template
- Each segment can have different styles
- Works with any split mode (char, word, line)
- Compatible with CSS animations and variables

**Developer experience:**
- Familiar HTML template syntax
- Easy to prototype and iterate
- Works with React components (auto-wrapped)
- Clear separation of structure and content

**Code example:**
```html
<ef-text split="char">
  <template>
    <ef-text-segment class="shadow" />
    <ef-text-segment class="main" />
  </template>
  Hello
</ef-text>
```

This declarative approach creates layered effects without JavaScript, with all segments automatically synchronized.
