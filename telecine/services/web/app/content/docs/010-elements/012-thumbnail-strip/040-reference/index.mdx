## Reference

### Grouped Attributes

#### Targeting

<PropertyDocList>
  <PropertyDoc name="target" type="string" domReadable domWritable htmlAttribute="target">
    The ID of the video element to generate thumbnails from. Do not include the # character.

    The thumbnail strip automatically tracks the target video element and updates thumbnails when the video's properties change (trimStart, trimEnd, sourceIn, sourceOut, src).

    <div className="bg-blue-100 dark:bg-blue-900/30 text-blue-900 dark:text-blue-100 p-2 rounded border border-blue-200 dark:border-blue-800">
      💡 Specify the element ID without the # prefix (e.g., "my-video" to target a video with id="my-video").
    </div>
  </PropertyDoc>
</PropertyDocList>

#### Layout

<PropertyDocList>
  <PropertyDoc name="thumbnailWidth" type="number" domReadable domWritable htmlAttribute="thumbnail-width" defaultValue="80">
    The desired width of individual thumbnails in pixels.

    The thumbnail strip automatically calculates how many thumbnails to display based on the available width and this thumbnail width. Larger values result in fewer, more detailed thumbnails.

    Height is automatically calculated to maintain the video's aspect ratio within the strip's height. The strip uses a ResizeObserver to track container width and updates the thumbnail layout automatically when the container resizes.

    <div className="bg-blue-100 dark:bg-blue-900/30 text-blue-900 dark:text-blue-100 p-2 rounded border border-blue-200 dark:border-blue-800">
      💡 The number of thumbnails displayed is calculated as: `floor(stripWidth / (thumbnailWidth + gap)) + 1`, ensuring the strip is always filled.
    </div>
  </PropertyDoc>
</PropertyDocList>

#### Time Range

<PropertyDocList>
  <PropertyDoc name="startTimeMs" type="number" domReadable domWritable htmlAttribute="start-time-ms">
    Custom start time in milliseconds for thumbnail generation.

    **In trimmed mode (default):** `startTimeMs` is relative to the trimmed timeline. 0ms = start of trimmed portion (after trimStart/sourceIn). The value is automatically converted to source timeline coordinates for thumbnail generation.

    **In intrinsic mode (`useIntrinsicDuration="true"`):** `startTimeMs` is relative to the source timeline. 0ms = start of source media.

    When specified, overrides the natural start time of the video (including any trimming) to generate thumbnails from this time in the source media.

    For details on timeline coordinate handling, see the [Timeline Coordinates explanation](/docs/elements/thumbnail-strip/explanation/timeline-coordinates).
  </PropertyDoc>

  <PropertyDoc name="endTimeMs" type="number" domReadable domWritable htmlAttribute="end-time-ms">
    Custom end time in milliseconds for thumbnail generation.

    **In trimmed mode (default):** `endTimeMs` is relative to the trimmed timeline. The value is automatically converted to source timeline coordinates for thumbnail generation.

    **In intrinsic mode (`useIntrinsicDuration="true"`):** `endTimeMs` is relative to the source timeline.

    When specified, overrides the natural end time of the video (including any trimming) to generate thumbnails up to this time in the source media.

    Must be used in conjunction with `startTimeMs` to define a custom time range.

    For details on timeline coordinate handling, see the [Timeline Coordinates explanation](/docs/elements/thumbnail-strip/explanation/timeline-coordinates).
  </PropertyDoc>

  <PropertyDoc name="useIntrinsicDuration" type="boolean" domReadable domWritable htmlAttribute="use-intrinsic-duration" defaultValue="false">
    Use the full source media duration instead of the trimmed duration for thumbnail generation.

    When `true`, thumbnails are generated from the entire source video file, ignoring any `trimStart`, `trimEnd`, `sourceIn`, or `sourceOut` properties on the target video element.

    <div className="bg-orange-100 dark:bg-orange-900/30 text-orange-900 dark:text-orange-100 p-2 rounded border border-orange-200 dark:border-orange-800">
      ⚠️ This property is ignored if custom `startTimeMs` or `endTimeMs` values are provided. Custom time ranges take precedence over `useIntrinsicDuration`.
    </div>

    For details on timeline coordinate handling, see the [Timeline Coordinates explanation](/docs/elements/thumbnail-strip/explanation/timeline-coordinates).
  </PropertyDoc>
</PropertyDocList>
