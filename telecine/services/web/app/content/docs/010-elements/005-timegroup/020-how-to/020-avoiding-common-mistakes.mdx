## Avoid Common Timegroup Mistakes

### Time Coordinate Mistakes

**Wrong**: `childTimegroup.currentTime = 5` or expecting `currentTime = 1` for Child B in sequence  
**Right**: Write only to root: `rootTimegroup.currentTime = 5`. Read `ownCurrentTimeMs` for element-relative time. See [Time Propagation](/docs/elements/timegroup/explanation/time-propagation).

**Wrong**: Setting `currentTime` and immediately reading frame state  
**Right**: Use `await timegroup.seek(timeMs)` which waits for completion. See [Reference](/docs/elements/timegroup/reference).

### Mode and Attribute Confusion

**Wrong**: `fit="contain"` expecting duration to change  
**Right**: Use `mode="fit"` for duration inheritance, `fit="contain"` for visual scaling.

**Wrong**: `overlap="2s"` alone creates visual crossfade  
**Right**: Combine `overlap` with CSS animations using `--ef-transition-duration` variables. See [How to Use CSS Variables](/docs/elements/timegroup/how-to/css-variables).

### Configuration Mistakes

**Wrong**: `animation-delay: 3s` for fade-out  
**Right**: Use `animation-delay: var(--ef-transition-out-start)` or `calc(var(--ef-duration) - 2s)`.

**Wrong**: Setting different `fps` on parent (30) and children (60)  
**Right**: Set `fps` once on root timegroup. See [Frame Quantization](/docs/elements/timegroup/explanation/frame-quantization).
