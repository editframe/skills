## Timeline Coordinates: Flexible Time Reference Systems

The thumbnail strip supports two timeline coordinate systems - trimmed mode and intrinsic mode - allowing you to generate thumbnails relative to either the trimmed video portion or the full source media. This flexibility enables different use cases while maintaining consistent behavior.

### The Problem: Single Timeline System Is Limiting

Thumbnail systems that use a single timeline reference face limitations:

**Fixed reference point:**
- Thumbnails always relative to one timeline (usually source start)
- Cannot show thumbnails from trimmed portion separately
- Cannot show thumbnails from full source when video is trimmed
- Forces workarounds for different use cases

**Confusing coordinate conversion:**
- Developers must manually convert between timelines
- Error-prone calculations for trimmed vs source time
- Inconsistent behavior across different video configurations
- Difficult to reason about thumbnail positions

**Limited use case support:**
- Cannot build scrubbers for trimmed timeline
- Cannot preview full source when editing trimmed clips
- Forces preprocessing or manual coordinate management

### The Solution: Dual Timeline Modes with Automatic Conversion

The thumbnail strip provides two timeline modes with automatic coordinate conversion:

**Trimmed mode (default):**
- `startTimeMs` and `endTimeMs` relative to trimmed timeline
- 0ms = start of trimmed portion (after trimStart/sourceIn)
- Automatically converts to source timeline coordinates for thumbnail generation

**Intrinsic mode (`useIntrinsicDuration="true"`):**
- `startTimeMs` and `endTimeMs` relative to source timeline
- 0ms = start of source media (ignores trimming)
- Shows thumbnails from entire source video
- Useful for previewing full source while editing trimmed clips

**Automatic coordinate conversion:**
- Thumbnail strip handles timeline conversion internally
- No manual calculation required from developers
- Consistent behavior regardless of video trimming state

### Benefits for Your Application

**Flexible use cases:**
- Build scrubbers for trimmed timeline (trimmed mode)
- Preview full source while editing (intrinsic mode)
- Show custom time ranges with precise control
- Support different UI patterns without workarounds

**Developer experience:**
- No manual coordinate conversion required
- Clear semantic meaning for each mode
- Consistent behavior across configurations
- Reduced cognitive load when working with timelines

**User experience:**
- Thumbnails match what users see in playback (trimmed mode)
- Preview full source when needed (intrinsic mode)
- Smooth transitions between modes
- Intuitive thumbnail positioning
