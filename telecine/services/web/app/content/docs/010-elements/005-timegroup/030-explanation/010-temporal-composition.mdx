## Temporal Composition: Declarative Timeline Management

The timegroup element implements **temporal composition**: a declarative system where parent-child relationships automatically compute precise timing for every element in a video timeline. This approach significantly reduces manual timeline calculations and provides automatic updates when compositions change.

### The Problem: Manual Timeline Arrangement Presents Engineering Challenges

Using native platform features for video editing presents these engineering challenges:

**Manual position calculation:** Start times typically require calculation for each element, and changing one duration requires recalculating all subsequent positions. Complex nested timelines require extensive manual math.

**Fragile updates:** Modifying a single element can break the entire timeline with no automatic propagation of timing changes, making timing errors difficult to debug.

**Imperative timeline manipulation:** Code like `item2.startTime = item1.endTime` creates tight coupling, requiring updates in multiple places when durations change.

### The Solution: Declarative Parent-Child Relationships

Timegroups use declarative composition modes that automatically compute timing:

**Fixed mode**: Explicit duration (base case)  
**Sequence mode**: Automatic serial arrangement - children play one after another  
**Contain mode**: Automatic parallel arrangement - children play simultaneously  
**Fit mode**: Automatic duration inheritance - commonly used for backgrounds

### Benefits: Automatic Timeline Updates

Parent-child relationships automatically compute all timing—changing a child's duration updates the timeline without manual calculations. The four composition modes cover common patterns and can be combined for complex timelines:

```html
<ef-timegroup mode="sequence">
  <ef-timegroup mode="fixed" duration="2s">Item 1</ef-timegroup>
  <ef-timegroup mode="fixed" duration="3s">Item 2</ef-timegroup>
</ef-timegroup>
```

Changing Item 1's duration from `2s` to `5s` automatically shifts Item 2's start time and updates the total duration from 5s to 8s, eliminating the need for manual position updates like `item2.startTime = item1.endTime`.
