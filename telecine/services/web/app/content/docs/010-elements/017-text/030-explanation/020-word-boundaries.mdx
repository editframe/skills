## Word Boundary Preservation: Preventing Line Breaks in Character Mode

When splitting text by characters, the text element automatically detects word boundaries and wraps characters within words to prevent line breaks. This ensures readable text layout while enabling character-by-character animations.

### The Problem: Character Splitting Breaks Word Layout

When text is split into individual characters, each character becomes a separate element:

**Layout problems:**
- Characters can break across lines mid-word
- Words become unreadable when split across lines
- Text wrapping becomes unpredictable
- Visual appearance is broken

**Example:** "Hello world" split by character might render as:
```
Hel
lo 
wor
ld
```

Instead of:
```
Hello
world
```

### The Solution: Word Boundary Detection and Wrapping

The text element uses `Intl.Segmenter` to detect word boundaries and wraps characters within words:

**Word detection:**
- Uses `Intl.Segmenter` with `granularity: "word"` to identify word boundaries
- Creates a map from character index to word index
- Characters within the same word share the same word index

**Word wrapping:**
- Characters within words are wrapped in `<span class="ef-word-wrapper">` elements
- Word wrapper has `display: inline-block` and `white-space: nowrap`
- Prevents line breaks within words
- Allows normal text wrapping between words

**How it works:**
1. Text is split into characters using grapheme-aware segmentation
2. Word boundaries are detected using `Intl.Segmenter`
3. Characters are grouped by word index
4. Each word group is wrapped in a word wrapper span
5. Words can wrap to new lines, but characters within words stay together

### Benefits

**Readable text:**
- Words never break across lines
- Text maintains natural reading flow
- Visual appearance matches user expectations
- Works with any language that has word boundaries

**Character animations:**
- Each character is still a separate segment
- Can animate characters individually
- Stagger timing works per character
- Template pattern works with individual characters

**Automatic handling:**
- No manual word detection needed
- Works with any language the browser supports
- Handles punctuation and whitespace correctly
- No configuration required

**Performance:**
- Single pass through text for word detection
- Minimal DOM overhead (one wrapper per word)
- Efficient browser-native segmentation
- Scales well with long text

**Cross-language support:**
- Works with languages that have word boundaries
- Handles mixed-language text correctly
- Respects locale-specific word segmentation rules

**Code example:**
```html
<ef-text split="char" stagger="50ms">
  <template>
    <ef-text-segment />
  </template>
  Hello world
</ef-text>
```

Characters within words stay together on the same line, while words can wrap normally. This enables character-by-character animations without breaking text readability.
