## Reference

### Grouped Attributes

#### Media Source

<PropertyDocList>
  <PropertyDoc name="assetId" type="string" domReadable domWritable htmlAttribute="asset-id">
    The asset ID of the audio.

    An asset ID is a unique identifier for a media file that has been uploaded to Editframe. This is documented in the [Processing Files](/docs/processing-files/audio-video-files) section.
  </PropertyDoc>

  <PropertyDoc name="src" type="string" domReadable domWritable htmlAttribute="src">
    The source URL/path of the audio.

    <div className="bg-blue-100 dark:bg-blue-900/30 text-blue-900 dark:text-blue-100 p-2 mb-2 rounded border border-blue-200 dark:border-blue-800">
      ⚠️ This property is not yet supported in all circumstances. Providing an arbitrary URL or path will not work unless the server is configured to respond with a precise, not-yet-documented, format.

      Demonstration projects with such a server are available via `npm create @editframe@beta`. These are not intended for production use. When complete, a specialized server should not be required, though there will be specific encoding/muxing limitations.

      Instead upload media to Editframe and use the `assetId` property.
    </div>

    <div className="bg-orange-100 dark:bg-orange-900/30 text-orange-900 dark:text-orange-100 p-2 rounded border border-orange-200 dark:border-orange-800">
      🚫 This property should only be used for development, testing, and previewing media to end-users in real-time. When submitting a render job, all media should be uploaded to Editframe and the `assetId` property should be used. Our render servers will render in parallel, and in order to operate efficiently we need to be able to load only the slice of media that is needed to render a small segment of the timeline.
    </div>
  </PropertyDoc>
</PropertyDocList>

#### Time Coordinates

<PropertyDocList>
  <PropertyDoc name="startTimeMs" type="number" domReadable>
    The start time of the audio in milliseconds. This time is relative to the timeline described by the root timegroup the audio is contained within, or 0 if the audio is standalone (no timegroup).
  </PropertyDoc>

  <PropertyDoc name="endTimeMs" type="number" domReadable>
    The end time of the audio in milliseconds. This time is relative to the timeline described by the root timegroup the audio is contained within, or the audio's duration if standalone (no timegroup).
  </PropertyDoc>

  <PropertyDoc name="durationMs" type="number" domReadable>
    The duration of the audio in milliseconds. This is equivalent to the difference between `endTimeMs` and `startTimeMs`.
  </PropertyDoc>

  <PropertyDoc name="currentTimeMs" type="number" domReadable>
    The current time of the audio in milliseconds. This time is the time within the timeline of the root timegroup the audio is contained within, or the audio's own timeline if standalone (no timegroup).
  </PropertyDoc>

  <PropertyDoc name="ownCurrentTimeMs" type="number" domReadable>
    The current time of the audio in milliseconds. This time is scoped directly to the audio element itself.
  </PropertyDoc>

  <PropertyDoc name="currentSourceTimeMs" type="number" domReadable>
    This property is the current time of the audio in milliseconds scoped to the source media. If no `trimStart` or `sourceIn` are set, this property is equivalent to `ownCurrentTimeMs`.

    <div className="bg-blue-100 dark:bg-blue-900/30 text-blue-900 dark:text-blue-100 p-2 mb-2 rounded border border-blue-200 dark:border-blue-800">
      💡 If you want to associate data to an audio clip, but it's important that the data is not affected by trimming, use this property. For example, if you were tracking audio events in the audio, you should use this time, so that tracking would hold true even if a `trimStart` value is set later.
    </div>
  </PropertyDoc>
</PropertyDocList>

#### Trimming & Editing

<PropertyDocList>
  <PropertyDoc name="trimStart" type="timestring" domReadable domWritable htmlAttribute="trimstart">
    A duration of time to trim off from the start of the audio.

    This time is relative to the start of the audio.

    Setting trimStart to 10s will result in the audio starting 10 seconds into the source media.

    This property is intended to be used with the `trimEnd` property.
  </PropertyDoc>

  <PropertyDoc name="trimEnd" type="timestring" domReadable domWritable htmlAttribute="trimend">
    A duration of time to trim off from the end of the audio.

    This time is relative to the end of the audio.

    Setting trimEnd to 10s will result in the audio ending 10 seconds into the source media.

    This property is intended to be used with the `trimStart`.
  </PropertyDoc>

  <PropertyDoc name="sourceIn" type="timestring" domReadable domWritable htmlAttribute="sourcein">
    A duration of time to trim off from the start of the audio.

    This time is an absolute value in the timeline of the source media.

    Setting sourceIn to 10s will result in the audio starting 10 seconds into the source media.

    This property is intended to be used with the `sourceOut` property.
  </PropertyDoc>
  <PropertyDoc name="sourceOut" type="timestring" domReadable domWritable htmlAttribute="sourceout">
    A duration of time to trim off from the end of the audio.

    This time is an absolute value in the timeline of the source media.

    Setting sourceOut to 10s will result in the audio ending 10 seconds into the source media.

    This property is intended to be used with the `sourceIn` property.
  </PropertyDoc>
</PropertyDocList>

#### Hierarchy

<PropertyDocList>
  <PropertyDoc name="parentTimegroup" type="<ef-timegroup> | null" domReadable>
    The closest timegroup element that contains the audio, or `null` if the audio is not contained within any timegroup (standalone audio).
  </PropertyDoc>

  <PropertyDoc name="rootTimegroup" type="<ef-timegroup> | null" domReadable>
    The outermost timegroup element that contains the audio, or `null` if the audio is standalone and not part of any composition.
  </PropertyDoc>
</PropertyDocList>

#### Audio Analysis

<PropertyDocList>
  <PropertyDoc name="fftSize" type="number" domReadable domWritable htmlAttribute="fft-size" defaultValue="512">
    The size of the FFT (Fast Fourier Transform) to use for audio analysis.

    This property is used when associating the audio with a waveform or other audio analysis.

    The value MUST be a power of 2. Higher values will result in more granular waveforms. Though this is not always more aesthetically pleasing.
  </PropertyDoc>

  <PropertyDoc name="fftDecay" type="number" domReadable domWritable htmlAttribute="fft-decay" defaultValue="8">
    The decay of the FFT (Fast Fourier Transform) to use for audio analysis.

    To create a smoother waveform animation, we average together several frames worth of audio data. This property controls the number of frames to average.

    Higher values will result in a smoother animation. Extremely high values will increase computational cost.
  </PropertyDoc>

  <PropertyDoc name="fftGain" type="number" domReadable domWritable htmlAttribute="fft-gain" defaultValue="3">
    The gain of the FFT (Fast Fourier Transform) to use for audio analysis.

    This property is used when associating the audio with a waveform or other audio analysis.

    Higher values will result in a more pronounced waveform.
  </PropertyDoc>

  <PropertyDoc name="interpolateFrequencies" type="boolean" htmlAttribute="interpolate-frequencies" defaultValue="false">
    Some input audio may have been fed through a highpass, resulting in waveform visualizations that appear to be missing data.

    For example, if audio was recorded through a telephone network, which typically cuts out frequencies above ~4khz, the waveform will appear to be missing data above this frequency.

    Setting this property will ignore zero values on the high end of the frequency spectrum, and interpolate the data to fill in the missing values.

    This will result in a more aesthetically pleasing waveform, but it may also result in a loss of precision.
  </PropertyDoc>

  <PropertyDoc name="mute" type="boolean" domReadable domWritable htmlAttribute="mute" defaultValue="false">
    Mutes the audio.

    When set to true, the audio will not produce any sound. Note that any waveforms connected to the media will continue to produce visual output, as mute only affects audio output via speakers.
  </PropertyDoc>
</PropertyDocList>
