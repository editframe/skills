# Test Reports

This directory contains test reports generated by the `test-runner` subagent.

## File Structure

For each test run, two files are created:

- `{timestamp}-report.md` - Markdown summary with failures and recommendations
- `{timestamp}-output.log` - Complete, untruncated test output

## Usage

**test-failure-fixer** reads these files to diagnose issues without re-running tests:
1. Start with the report file (`.md`) for summary
2. Read/grep the log file (`.log`) for detailed analysis

## Gitignore

Both `.md` and `.log` files are gitignored (except this README).
