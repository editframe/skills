## Frame Quantization: Frame-Perfect Synchronization

To ensure that animations and media playback are perfectly synchronized, `ef-timegroup` quantizes all time updates to a consistent frame grid. This eliminates floating-point drift and ensures predictable, smooth output especially in rendering environments.

### The Problem: Floating-Point Time Values Present Engineering Challenges

Using floating-point time values directly presents these engineering challenges:

**Floating-point precision errors:** Time values like `1.234567890123` don't align with frame boundaries, causing rounding errors that accumulate and drift animations out of sync.

**Frame misalignment:** Setting `currentTime = 1.234` might land between frames, causing different elements to interpret the same time differently and inconsistent behavior between playback and rendering.

**Seeking unpredictability:** Seeking to the same time twice might land on different frames, making frame state non-deterministic and timing issues hard to debug.

### The Solution: Quantization to Frame Boundaries

Timegroups automatically quantize all time values to frame boundaries based on the configured `fps`:

```
quantized_time = round(time × fps) / fps
```

All time values snap to exact frame boundaries—setting `currentTime = 1.234` with `fps=30` becomes `1.233...` (37th frame). Same time value always lands on same frame, ensuring playback and rendering produce identical results with deterministic frame state.

### Benefits for Your Workflow

Every frame is exactly where it should be, ensuring consistent output between preview and final render. Seeking always lands on exact frames, providing reliable frame state for programmatic control and deterministic system behavior.

### How Users Experience These Benefits

Setting `currentTime` automatically quantizes to the nearest frame boundary:

```javascript
const timegroup = document.getElementById('my-video');
timegroup.fps = 30;
timegroup.currentTime = 1.234; // Automatically quantizes to frame 37 (1.233...s)
```

With `fps=30`, setting `currentTime = 1.234` quantizes to frame 37 (`1.233...s`), ensuring all elements show frame 37's state consistently.
