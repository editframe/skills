## Create Karaoke-Style Captions

Create progressive word highlighting where completed words are styled differently from the active word, similar to karaoke displays.

### Solution

Combine `<ef-captions-before-active-word>`, `<ef-captions-active-word>`, and `<ef-captions-after-active-word>` with CSS gradients:

```html
<ef-captions captions-script="karaoke-captions" className="text-center">
  <ef-captions-before-active-word className="text-yellow-300 text-2xl font-bold" />
  <ef-captions-active-word className="text-white text-2xl font-bold karaoke-word" />
  <ef-captions-after-active-word className="text-gray-400 text-2xl font-bold" />
</ef-captions>
```

```css
.karaoke-word {
  background: linear-gradient(
    90deg,
    #fbbf24 0%,
    #f59e0b calc(var(--ef-progress) * 100%),
    #ffffff calc(var(--ef-progress) * 100%),
    #ffffff 100%
  );
  background-clip: text;
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
  animation: karaoke-fill calc(var(--ef-duration) * 1ms) linear;
}

@keyframes karaoke-fill {
  0% {
    background: linear-gradient(90deg, #fbbf24 0%, #fbbf24 0%, #ffffff 0%, #ffffff 100%);
  }
  100% {
    background: linear-gradient(90deg, #fbbf24 0%, #fbbf24 100%, #ffffff 100%, #ffffff 100%);
  }
}
```

### How It Works

- **Before words**: Styled in yellow (completed)
- **Active word**: Uses gradient fill that progresses from yellow to white using `--ef-progress`
- **After words**: Styled in gray (upcoming)

The `--ef-progress` variable is a number (0-1), not a CSS percentage. It tracks the current word's progress, enabling smooth fill animations. When used in gradient stops or other places that require a percentage, multiply by 100% using `calc(var(--ef-progress) * 100%)`. This format ensures it works correctly in CSS calculations without being converted to pixel values based on element dimensions.
