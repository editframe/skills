## Display Multiple Surfaces from a Single Source

**Problem:** You need to display the same video multiple times in your composition, but using multiple video elements would be inefficient and resource-intensive.

**Solution:** Use multiple surface elements that all target the same video element. Each surface mirrors the video's canvas content, sharing a single video decode and network stream.

<Demonstration layout="horizontal" sourceVisible alwaysShowSource enablePreview enableFilmstrip enableFitScale>
  <Timegroup mode="contain" className="w-[720px] h-[480px] flex items-center justify-center bg-black">
    <Video id="main-video" src="https://assets.editframe.com/bars-n-tone.mp4" className="w-1/3 h-full object-contain" />
    <Surface target="main-video" className="w-1/3" />
    <Surface target="main-video" className="w-1/3" />
  </Timegroup>
</Demonstration>

**Key points:**
- The target video element must have an `id` attribute
- All surfaces reference the same `id` in their `target` attribute
- Surfaces automatically stay synchronized with the target video
- Each surface can have different CSS styling applied

For deeper understanding of why this approach is more efficient, see the [Canvas Mirroring Architecture explanation](/docs/elements/surface/explanation/canvas-mirroring-architecture).
