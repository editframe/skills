## Aspect Ratio Preservation: Scaling Without Distortion

ef-fit-scale addresses the challenge of displaying fixed-size compositions responsively while maintaining their aspect ratio. This enables compositions designed for specific output dimensions (like 1920x1080) to adapt to any display size without distortion or cropping.

### Engineering Challenges with Container Scaling

Using CSS alone to scale entire composition trees presents these constraints: CSS `object-fit` and `background-size` work for individual elements but don't scale entire DOM subtrees. Transform scaling requires manual calculation of scale factors and centering offsets. Responsive layouts typically require media queries and multiple composition variants, increasing complexity.

The core challenge is scaling an entire composition tree (timegroup with nested elements) as a unit while maintaining proportions and ensuring it fits entirely within the container.

### The Solution: Continuous Aspect-Ratio-Aware Scaling

ef-fit-scale calculates the optimal scale factor by comparing container and content aspect ratios. It selects the scale that fits the content entirely within the container (contain behavior), then centers the scaled content using CSS transforms. Updates occur continuously via requestAnimationFrame, ensuring smooth scaling as container dimensions change.

**Architectural advantages:**
- **Maintains composition integrity:** The entire composition scales as a unit, preserving relationships between nested elements
- **No distortion:** Aspect ratio is preserved, preventing stretching or squashing
- **No cropping:** Content fits entirely within the container, ensuring nothing is cut off
- **Smooth updates:** Frame-synced updates provide responsive scaling as containers resize
- **Works with any content:** Not limited to video—works with any composition structure

### Benefits for Your Workflow

**Responsive design:** Create compositions at fixed output dimensions (1920x1080, 1080x1920, etc.) and display them responsively without creating multiple variants. One composition adapts to mobile, tablet, and desktop screens.

**Consistent appearance:** Aspect ratio preservation ensures compositions look correct across all screen sizes. No distortion means text remains readable and visual elements maintain their intended proportions.

**Simplified development:** No need to calculate scale factors manually or write media queries for different screen sizes. Wrap the composition in ef-fit-scale and it adapts automatically.

**Performance:** Continuous updates are optimized via requestAnimationFrame, providing smooth scaling without performance overhead.

### Example: Responsive Composition

Wrap any fixed-size composition in ef-fit-scale to make it responsive:

<Demonstration layout="horizontal" sourceVisible alwaysShowSource enablePreview enableFilmstrip enableFitScale>
  <ResizableContainer initialWidth={400} initialHeight={300}>
    <FitScale style={{ width: '100%', height: '100%' }}>
      <div style={{ width: '1920px', height: '1080px', background: 'linear-gradient(135deg, #667eea 0%, #764ba2 100%)', display: 'flex', alignItems: 'center', justifyContent: 'center', color: 'white', fontSize: '24px', fontWeight: 'bold' }}>
        1920 × 1080
      </div>
    </FitScale>
  </ResizableContainer>
</Demonstration>

```jsx
<FitScale style={{ width: '100%', height: '400px' }}>
  <Timegroup mode="contain" className="w-[1920px] h-[1080px]">
    {/* Your composition */}
  </Timegroup>
</FitScale>
```

Resize the container—the composition scales while maintaining its 16:9 aspect ratio. The scale factor is calculated automatically, and the content is centered within the container.

ef-fit-scale finds the first content child element (excluding style, script, and other non-content elements) and scales it. This enables flexible composition structures while ensuring predictable scaling behavior.
