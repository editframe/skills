## Captions Data Format: Dual-Level Timing Architecture

Editframe captions use a dual-level timing structure with both `segments` (phrase-level) and `word_segments` (word-level) timing data. This architecture enables flexible caption displays while maintaining precise synchronization and efficient rendering.

### The Problem: Single-Level Timing Limitations

Using native caption systems with only one level of timing granularity presents these constraints:

**Phrase-only systems:**
- Display complete sentences at once
- Cannot highlight individual words as they're spoken
- Limited animation possibilities
- Poor support for karaoke-style or word-by-word displays

**Word-only systems:**
- Require complex logic to group words into readable phrases
- Difficult to display complete thoughts or sentences
- May break natural reading flow
- Performance overhead from processing every word individually

**Fixed-format systems:**
- Locked into a single display style
- Cannot adapt to different use cases (accessibility, karaoke, educational)
- Require format conversion for different display needs

### The Solution: Dual-Level Timing Structure

Editframe's captions use both `segments` and `word_segments` arrays, each serving distinct purposes:

**Segments (phrase-level timing):**
- Complete phrases, sentences, or thought units
- Natural reading boundaries
- Efficient for displaying full caption text
- Enables context-aware displays

**Word segments (word-level timing):**
- Individual word timing with precise start/end boundaries
- Enables word-by-word highlighting and animations
- Supports karaoke-style progressive displays
- Allows varied styling per word

### Architectural Advantages

**Flexible display modes:**
- **Segment display**: Show complete phrases (`<ef-captions-segment>`)
- **Word display**: Highlight individual words (`<ef-captions-active-word>`)
- **Context display**: Show before/active/after words for reading flow
- **Hybrid displays**: Combine segment and word timing for rich experiences

**Efficient rendering:**
- Segments provide natural grouping boundaries, reducing DOM updates
- Word segments enable precise timing without processing overhead
- System only updates elements when timing boundaries are crossed
- Minimal re-renders compared to processing every character or syllable

**Precise synchronization:**
- Word-level timing ensures accurate highlighting even for fast speech
- Segment-level timing provides natural reading boundaries
- Both levels update simultaneously, maintaining consistency
- Supports frame-accurate caption display

**Developer experience:**
- Single data structure supports multiple display patterns
- No format conversion needed between different use cases
- Child elements automatically receive appropriate timing data
- Simple API: provide both arrays, use what you need

### Code Examples

**Using segments for full phrase display:**
```html
<ef-captions captions-script="my-captions">
  <ef-captions-segment />
</ef-captions>
```

**Using word_segments for word-by-word highlighting:**
```html
<ef-captions captions-script="my-captions">
  <ef-captions-active-word />
</ef-captions>
```

**Combining both for context display:**
```html
<ef-captions captions-script="my-captions">
  <ef-captions-before-active-word />
  <ef-captions-active-word />
  <ef-captions-after-active-word />
</ef-captions>
```
