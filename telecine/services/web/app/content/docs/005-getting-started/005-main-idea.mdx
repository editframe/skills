# What's the general idea?

<div className="not-prose bg-blue-50 dark:bg-blue-950/30 border-l-4 border-blue-500 p-4 my-6 rounded-r">
  <div className="text-lg font-semibold text-blue-900 dark:text-blue-100 mb-2">
    If you can get it to show up in a browser, we'll render it!
  </div>
  <div className="text-blue-800 dark:text-blue-200">
    Editframe renders HTML, CSS, and JavaScript into videos. Build your video composition using familiar web technologies, and we'll handle the rendering.
  </div>
</div>

<div className="not-prose flex items-center gap-4">
  <div>
    <div>The fundamental representation of a video in editframe is an HTML document.</div>

    <div>This HTML document is used to define the timeline, the visual presentation of the video, and timing of animations.</div>

    <div>Used together HTML, CSS, and JavaScript comprise a powerful compositing system.</div>
  </div>

  <img src="/assets/overall-picture.png" className="w-1/2" alt="Iconographic depiction of html source code transforming into a video timeline, stacked layers, and an animating disc." />
</div>

<div className="not-prose flex flex-col items-center">
  <div className="w-1/2">
    <div>Fundamental editing and compositing primitives such as cutting, sequencing, stacking, and keyframing can all be expressed with HTML, CSS, and JavaScript.</div>

    <div>Some are built-in, and the rest Editframe provides as custom elements.</div>
  </div>

  <img src="/assets/editing-primitives.png" className="w-3/4" alt="Iconographic depiction of cutting a filmstrip, stacking visual layers, and setting keyframe animations." />
</div>

<div className="not-prose flex flex-col items-center">
  <div className="w-1/2">
    <div>The advantages of using HTML, CSS, and JavaScript are many:</div>

    <ul>
      <li>Use familiar HTML, CSS, and JS skills</li>
      <li>Reuse existing code for consistent styling</li>
      <li>Leverage modern web platform features</li>
      <li>Access extensive tooling and libraries</li>
    </ul>
  </div>

  <img src="/assets/advantage.png" className="w-1/2" alt="Iconographic depiction a computer with html/css/js" />
</div>

<div className="not-prose">
  <div>Editframe uses a set of custom HTML elements to define video timelines, their visual presentation, and render them into videos.</div>

  <div>It's not terribly difficult to create your own videos with Editframe, but there are a few fundamental concepts you'll need to understand. But once you've mastered the basic elements, you'll be able to apply your existing knowledge of web technologies to create any video your heart desires.</div>

  <div>You can use the following elements to create a video:</div>

  <ul>
    <li><a href="/docs/elements/timegroup">Timegroup</a> - A container for other elements (start here)</li>
    <li><a href="/docs/elements/image">Image</a> - Include an image in your video</li>
    <li><a href="/docs/elements/video">Video</a> - Include a video in your video</li>
    <li><a href="/docs/elements/audio">Audio</a> - Include audio file in your video</li>
    <li><a href="/docs/elements/waveform">Waveform</a> - A visual representation of an audio file</li>
  </ul>
</div>
