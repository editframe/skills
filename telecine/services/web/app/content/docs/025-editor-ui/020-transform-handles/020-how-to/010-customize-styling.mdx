## Customize Handle Styling

**Problem:** You need to customize the colors and appearance of transform handles to match your design system.

**Solution:** Use CSS custom properties to override default handle styles.

<Demonstration layout="horizontal" sourceVisible alwaysShowSource enableFitScale>
  <style dangerouslySetInnerHTML={{ __html: `
    ef-transform-handles {
      --ef-transform-handles-border-color: #8b5cf6;
      --ef-transform-handles-dragging-border-color: #7c3aed;
      --ef-transform-handles-handle-color: #f3f4f6;
      --ef-transform-handles-handle-border-color: #8b5cf6;
      --ef-transform-handles-rotate-handle-color: #10b981;
    }
  ` }} />
  <div style={{ position: 'relative', width: '100%', height: '400px', background: '#f3f4f6', border: '2px solid #e5e7eb' }}>
    <div id="element-styling" style={{ position: 'absolute', left: '100px', top: '100px', width: '200px', height: '150px', background: '#8b5cf6', borderRadius: '8px' }}></div>
    <TransformHandles target="#element-styling" enableRotation />
  </div>
</Demonstration>

**Available CSS variables:**
- `--ef-transform-handles-border-color`: Border color of the overlay (default: #3b82f6)
- `--ef-transform-handles-dragging-border-color`: Border color when dragging (default: #2563eb)
- `--ef-transform-handles-handle-color`: Background color of resize handles (default: white)
- `--ef-transform-handles-handle-border-color`: Border color of resize handles (default: #3b82f6)
- `--ef-transform-handles-rotate-handle-color`: Background color of rotation handle (default: #10b981)

Apply these variables to the `ef-transform-handles` element or a parent container to style all handles consistently.
