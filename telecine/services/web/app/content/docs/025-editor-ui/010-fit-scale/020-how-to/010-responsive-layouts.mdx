## Create Responsive Layouts

**Problem:** You have a fixed-size composition (e.g., 1920x1080) that needs to adapt to different screen sizes while maintaining aspect ratio.

**Solution:** Wrap your composition in `<ef-fit-scale>` with a responsive container.

<Demonstration layout="horizontal" sourceVisible alwaysShowSource>
  <ResizableContainer initialWidth={400} initialHeight={300}>
    <FitScale style={{ width: '100%', height: '100%' }}>
      <div style={{ width: '1920px', height: '1080px', background: 'linear-gradient(135deg, #667eea 0%, #764ba2 100%)', display: 'flex', alignItems: 'center', justifyContent: 'center', color: 'white', fontSize: '24px', fontWeight: 'bold' }}>
        1920 × 1080
      </div>
    </FitScale>
  </ResizableContainer>
</Demonstration>

**HTML structure:**

```html
<div class="responsive-container">
  <ef-fit-scale style="width: 100%; height: 400px;">
    <ef-timegroup mode="contain" class="w-[1920px] h-[1080px]">
      <ef-video src="video.mp4" class="w-full h-full"></ef-video>
    </ef-timegroup>
  </ef-fit-scale>
</div>
```

**CSS for responsive container:**

```css
.responsive-container {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
}
```

**Different aspect ratios:**

- **16:9 (HD):** `w-[1920px] h-[1080px]`
- **9:16 (Vertical):** `w-[1080px] h-[1920px]`
- **1:1 (Square):** `w-[1080px] h-[1080px]`
- **4:5 (Instagram):** `w-[1080px] h-[1350px]`

The composition scales to fit the container while preserving its aspect ratio. See [Aspect Ratio Preservation](/docs/editor-ui/fit-scale/explanation/aspect-ratio-preservation) for how this works.
