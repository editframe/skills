# Introduction

The `<ef-waveform>` element is a custom component designed for use within the Editframe ecosystem. It provides a visual representation of an audio waveform, enhancing the audio playback experience.

## Key Features

* **Audio Visualization**: Displays a graphical representation of the audio waveform.
* **Sync with Audio**: Can be linked to an audio element to visualize the current playback position.
* **Customizable**: Allows for styling to match your project's design.

## Basic Usage

This example demonstrates the basic usage of the `<ef-waveform>` element.

<Playground 
presetCode={`
<ef-timegroup
  mode="sequence"
  class="w-[500px] h-[500px] bg-black relative overflow-hidden"
>
  <ef-timegroup mode="contain" class="flex flex-col items-center justify-center">
    {{code}}
  </ef-timegroup>
</ef-timegroup>
`}
code={`<ef-audio id="sample-audio" src="assets/audio-sample.mp3"></ef-audio>
<ef-waveform
  class="h-full w-full fill-green-600 stroke-athens-gray-500"
  target="sample-audio"
></ef-waveform>`} />

## Attributes

* **target**: The ID of the audio element to visualize.
* **mode**: The visualization mode for the waveform. Default is "bars".

This will create a waveform visualization for the audio with the ID "sample-audio". The waveform will be 12 units high and span the full width of its container.

## Styling

You can customize the appearance of the waveform using CSS classes. The `class` attribute allows you to apply custom styles to match your project's design.

## Use Cases

The `<ef-waveform>` element is particularly useful in the following scenarios:

* Audio players with visual feedback
* Music or podcast applications
* Audio editing interfaces
* Any project where visualizing audio content enhances the user experience

By using the `<ef-waveform>` element, you can create more engaging and interactive audio experiences within your Editframe projects.
